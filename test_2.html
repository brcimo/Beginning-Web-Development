Chapter 1: Introduction to the Internet and the World Wide Web

Active Learning Approach

Objective of this Lesson:

By the end of this lesson, you should be able to:

  1. Understand what the Internet is, how it evolved, and its key components.
  2. Explain the relationship between the Internet and the World Wide Web.
  3. Describe how web protocols, browsers, and domain names work together to deliver content.
  4. Understand the client-server model, the request-response cycle, and URLs.

Introduction:

Welcome to the first chapter of your web development journey! In this lesson, you will explore the foundational concepts of the Internet and the World Wide Web. We will cover the history of the Internet, how websites function, and how different components like browsers, servers, and protocols work together to deliver web content.

Lesson Content:

1. What is the Internet?

The Internet is a vast network of computers connected worldwide that communicate with each other. It enables data exchange, communication, and access to information on a global scale.

Key Milestones in the History of the Internet:

  • 1960s: The first ideas for the Internet began with the development of ARPANET, a research project funded by the U.S. government.
  • 1970s: Email was invented, and the first protocols for data exchange were established (like TCP/IP).
  • 1990s: The Internet became accessible to the public, and the World Wide Web (WWW) was created, revolutionizing how information was shared.
  • 2000s-present: The rise of social media, mobile access, and high-speed broadband further expanded the Internet’s reach.

Key Components of the Internet:

  • Servers: Computers that store data and serve it to other computers (clients) over the Internet.
  • Browsers: Software programs that allow users to access websites and view content. Examples: Google Chrome, Firefox, Safari.
  • Routers and Networks: Hardware and infrastructure that direct data between servers and clients.

2. The World Wide Web: A Layer Above the Internet

The World Wide Web (WWW) is a service that operates on the Internet, providing access to websites and web content. While the Internet is the infrastructure, the Web is a system of interlinked documents accessed via browsers.

Key Concepts:

  • Web Protocols (HTTP/HTTPS):
    • HTTP (Hypertext Transfer Protocol) is a protocol for transferring data across the web. It tells browsers how to request and receive web content.
    • HTTPS (Hypertext Transfer Protocol Secure) is the secure version of HTTP, encrypting data to protect privacy during transmission.
  • Websites, Webpages, and Domains:
    • Website: A collection of interconnected web pages.
    • Webpage: A single document within a website, typically a HTML file.
    • Domain Name: A human-readable address used to access websites (e.g., www.example.com).
  • Browsers and Their Role in Rendering Web Content:
    • Browsers interpret HTML, CSS, and JavaScript code to display websites to users. They retrieve files from servers and render them visually on your screen.

3. How Websites Work

A website involves multiple steps and systems to make sure you can view a page on your device. Let's break down the process:

  • Client-Server Model:
    • The client is your device (e.g., a computer or smartphone). The server is where the website's data is stored. When you request a webpage, your browser (the client) asks the server for the page.
  • Request-Response Cycle:
    1. Request: When you type a URL into the browser, the browser sends a request to the server hosting the website.
    2. Response: The server receives the request, processes it, and sends back the requested data (webpage, images, etc.).
    3. Render: The browser interprets the received data and renders the content visually on your screen.
  • URLs and Domain Names:
    • URL (Uniform Resource Locator): The web address you type into the browser's address bar. For example, https://www.example.com.
    • A Domain Name is part of the URL, and it represents the website’s address in a human-readable form. Behind the scenes, the domain name is translated into an IP address that servers use to communicate.

Interactive Exercises:

1. Internet & WWW Brainstorming Activity:

Objective: To help students understand the difference between the Internet and the Web.

Instructions:

  • Take a moment to think about the Internet and the Web. Write down your thoughts on the following questions:
    • What do you think the Internet is?
    • What is the World Wide Web? How is it different from the Internet?
    • How do you access websites on the Internet?

After a few minutes, discuss your answers with a partner or in a small group. Compare and contrast your definitions.

Reflection:

  • Discuss as a class the main differences. Highlight that the Internet is the physical infrastructure, while the World Wide Web is the system of documents accessible through the Internet.

2. URL Dissection Exercise:

Objective: To identify the parts of a URL and understand how browsers use it to retrieve information.

Instructions:
Below are three URLs. Break them down into their components (protocol, domain, path, etc.):

  1. https://www.google.com/search?q=web+development
  2. http://www.example.org/contact.html
  3. https://en.wikipedia.org/wiki/HTML

Task:

  • Identify the protocol (HTTP/HTTPS).
  • Identify the domain name.
  • Identify the path (the specific page or resource on the server).
  • Write down how the URL is structured.

Discussion:
After completing the task, review your findings with the class. Discuss the function of each part of the URL and how the browser uses them to find and display a webpage.

3. Client-Server Model Simulation:

Objective: To visualize how the client-server model works in real life.

Instructions:

  • Pair up with a classmate. One of you will act as the client (the person requesting the web page) and the other as the server (the one delivering the webpage).
  • The client will ask for a page (e.g., “Please give me the homepage of www.example.com”), and the server will respond by "sending" back the page (you can print out a page on paper or simply describe what happens).
  • Switch roles after a few rounds.

Reflection:
After the exercise, discuss the roles of the client and server in this interaction. Talk about how each step (request, response) mirrors what happens when you access a website.

Quiz: Test Your Understanding

  1. What is the main difference between the Internet and the World Wide Web?
  2. What does HTTP stand for, and why is HTTPS important?
  3. What are the three main components that make up a URL?
  4. In the client-server model, what is the role of the client and the role of the server?
  5. What happens when you type a URL into a browser?
